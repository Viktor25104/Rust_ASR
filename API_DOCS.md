# Документация WebSocket API (RustASR Server)

RustASR переделан в WebSocket микросервис для обработки потокового аудио в реальном времени.

## Подключение
- **URL:** `ws://<HOST>:<PORT>/transcribe`
- Настройки сервера задаются через переменные окружения (ENV) при запуске контейнера:
  - `MODEL_PATH` - Путь к весам модели (по-умолчанию `models/gigaam-v3-e2e-ctc`).
  - `MODEL_TYPE` - Тип загружаемой модели (`gigaam`, `whisper`, `qwen3`, `parakeet`).
  - `DEVICE` - Устройство для вычислений (`cpu`, `cuda`, `metal`).
  - `HOST` - IP и Порт сервера (по-умолчанию `0.0.0.0:8080`).

## Входящие данные (Клиент -> Сервер)

Сервер ожидает сообщения двух типов:
1. **Binary Message (Сырое аудио):**
   - Формат: `PCM 16-bit Float (LE)` или совместимые сырые байты.
   - Sample Rate: **16000 Hz**
   - Каналы: **Mono** (1 канал)
   - *Примечание: Аудио накапливается во внутреннем буфере сервера.*

2. **Text Message (Команды управления):**
   - Если клиент отправляет строку, содержащую `"finish"`, сервер прекращает накопление текущего чанка прогоняет весь накопленный буфер через модель распознавания, отправляя ответ обратно.
   - *При необходимости можно допилить отправку JSON с парамтерами транскрипции.*

## Исходящие данные (Сервер -> Клиент)

После команды `"finish"` и успешной обработки аудио, сервер отправляет обратно клиенту ответ в формате **JSON / Text Message**.

**Пример ответа:**
```json
{
  "text": "Привет мир это проверка распознавания",
  "language": "ru",
  "rtf": 0.05,
  "segments": [
    {
      "start": 0.0,
      "end": 2.5,
      "text": "Привет мир это проверка распознавания",
      "confidence": 0.98
    }
  ]
}
```

- `text` - Полный очищенный текст.
- `language` - Распознанный (или заданный жестко) язык.
- `rtf` - Значение Real-Time Factor (скорость обработки 1 секунды аудио).
- `segments` - Массив таймкодов (слов или предложений).

## Сборка и Публикация в GitHub Container Registry (ghcr.io)

### 1. Авторизация
```bash
echo $CR_PAT | docker login ghcr.io -u <YOUR_GITHUB_USERNAME> --password-stdin
```
*(Где `CR_PAT` — ваш персональный токен доступа Github с правами `write:packages`)*

### 2. Сборка образа (A40 / RTX 3000-4000)
```bash
docker build -t ghcr.io/<YOUR_GITHUB_USERNAME>/rust-asr:latest -f Dockerfile .
```

### 3. Публикация
```bash
docker push ghcr.io/<YOUR_GITHUB_USERNAME>/rust-asr:latest
```

## Развертывание Docker контейнера на production-сервере (для NVIDIA A40)

Для запуска образа на сервере с видеокартой **Nvidia A40**, вам потребуется установленный **NVIDIA Container Toolkit** (`nvidia-docker2`).

### Проброс портов (Networking)
Контейнер работает как простой HTTP-вебсервер, который умеет делать Upgrade соединения до WebSocket. 
- **TCP/HTTP порт**: `8080` (дефолтный порт внутри контейнера).
- *Если снаружи (в облаке/файрволе) или через Nginx/Traefik вы хотите другой порт (например, `80` или `443`), вы пробрасываете его через `-p ВНЕШНИЙ:8080`.*

### Команда запуска (Docker Run)

Ниже приведена полная и точная команда для запуска микросервиса на видеокарте A40 с использованием скачанного образа:

```bash
docker run -d \
  --name rustasr_gigaam \
  --restart unless-stopped \
  --gpus '"device=all"' \
  -p 8080:8080 \
  -e DEVICE="cuda" \
  -e MODEL_TYPE="gigaam" \
  -e MODEL_PATH="/app/models/gigaam-v3-e2e-ctc" \
  -e HOST="0.0.0.0:8080" \
  -e RUST_LOG="info" \
  ghcr.io/<YOUR_GITHUB_USERNAME>/rust-asr:latest
```

### Разбор флагом и параметров
* `-d` — Запустить контейнер в фоне (detached mode).
* `--name rustasr_gigaam` — Дать контейнеру понятное имя.
* `--restart unless-stopped` — Авто-рестарт при падении или перезагрузке сервера.
* `--gpus '"device=all"'` — **Самое важное для A40**. Этот флаг пробрасывает все доступные Nvidia GPU внутрь контейнера. Если на сервере несколько видеокарт, а нужна только одна конкретная (например первая), используйте `--gpus '"device=0"'`.
* `-p 8080:8080` — **Сетевой маппинг**.
  * Левая часть `8080` — это **внешний TCP-порт** на хост-сервере. Вы должны его открыть в Firewall (ufw, iptables или в Security Group AWS/Selectel).
  * Правая часть `8080` — порт **внутри** контейнера. Эту цифру менять не нужно.

### Переменные окружения (ENV), передаваемые в контейнер
* `DEVICE="cuda"` — Обязательно "cuda", чтобы RustASR начал считать матрицы на видеокарте A40, а не на процессоре.
* `MODEL_TYPE="gigaam"` — Сообщаем движку архитектуру модели.
* `MODEL_PATH="/app/models/gigaam-v3-e2e-ctc"` — В эту папку мы "запекли" конвертированные нами веса в предыдущем шаге сборки (safetensors).
* `HOST="0.0.0.0:8080"` — Связка IP:Порт, на которой сервер слушает соединения *изнутри* докера. `0.0.0.0` означает, что он готов принимать запросы с любых сетевых интерфейсов.
* `RUST_LOG="info"` — В консоль докера (доступно по `docker logs -f rustasr_gigaam`) будут писаться логи в формате INFO: подключения клиентов и общие ошибки.
